# 运行与维护机理

## 状态流转

- `raw`：只建立基本来源记录；
- `processed`：已结构化整理；
- `refined`：已更新 Wiki、索引和日志；
- `evergreen`：只能由使用者确认，AI 不得自动设置。

## 差异分类

检查工具将资料分为：已录入、新资料、重复副本、已更新资料、缺失资料和同名候选。后五项是自动化门禁的可观察输入。

## 维护与摄入分权

- 维护任务只有读取、检查、报告权限；
- 摄入任务只有在明确门禁通过时获得有限写入权限；
- 删除、移动、覆盖、合并、改名和处理冲突始终需要人工决定。

## 动态验证

验证期望数量来自当前清单，不使用“历史上曾经有多少份资料”作为常量。每条清单记录都必须能回到原始相对路径、SHA256、来源笔记和 Wiki 路由。
