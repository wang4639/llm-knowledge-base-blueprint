# 安全与隐私

## 公开仓库允许出现

- 通用规则、模板、参数化脚本；
- 纯虚构样例；
- GitHub 公共作者账号；
- 空配置示例和相对路径。

## 永不进入仓库

- API Key、Token、Cookie、私钥、密码和真实 `.env`；
- 使用者姓名、私人邮箱、账号资料和本机绝对路径；
- 原始资料、已摄入知识、真实清单、哈希和维护历史；
- 数据库、日志、缓存和运行沙盒；
- Horizon 实现、配置、密钥、Cookie、日报和自动化。

## 发布闸门

`tests/Test-PublicRelease.ps1` 同时检查文件名和文本内容，包括常见秘密格式、私钥头、认证字段、本机用户路径、内部控制面及被禁止的 Horizon 实现线索。扫描通过不等于数学意义上的零风险，因此发布前还要人工审阅 Git 暂存清单和 diff。

## 一旦怀疑密钥泄漏

不要只删除文件。第一步是在供应商后台撤销或轮换密钥，再检查使用记录和 Git 历史。任何疑似真实密钥都按已泄漏处理。
