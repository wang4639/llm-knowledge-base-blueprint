# 自动增量摄入运行规则

本规则只供专用“每周增量摄入”任务使用。只读维护任务不得执行本规则中的写入步骤。

## 门禁

先运行 `scripts/Check-Inbox.ps1`。只有同时满足以下条件才能继续：

- `NewFiles` 为 1–30；
- `DuplicateCopies = 0`；
- `UpdatedFiles = 0`；
- `MissingFiles = 0`；
- `SameNameCandidates = 0`。

新资料为 0 时不摄入，只运行动态验证。超过配置上限或出现任何其他差异时只报告。

## 单份条件

- 是配置允许扩展名的普通文件；
- 处理前后 SHA256 不变；
- 能映射到配置中的一个板块；
- 来源笔记目标不存在；
- 可完整读取且没有需要覆盖的冲突。

## 顺序

对相对路径稳定排序，最多处理配置上限：

1. 记录根目录 ID、相对路径、SHA256、字节数和风险；
2. 按来源笔记模板结构化，使用者观点字段留空；
3. 更新或新建 Wiki；
4. 检查问答卡影响关系；
5. 更新板块 index 并追加 log；
6. 用 `Register-IngestedSource.ps1` 追加清单；
7. 再次检查差异并运行动态验证；
8. 写入脱敏维护报告。

任一文件出现无法分类、读取失败、哈希变化、目标存在、内容冲突、注册失败或验证失败时，停止本次余下资料并报告已完成范围。原始资料全程只读。
