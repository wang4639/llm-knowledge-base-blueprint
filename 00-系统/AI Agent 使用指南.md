# AI Agent 使用指南

处理本知识库中的问答、写入或维护任务前，必须先阅读本文件，再阅读根目录 [[index|知识库总首页]]。

## 系统地图

- `10-来源笔记/`：单份资料的结构化记录与可回溯证据；不是默认问答入口。
- `20-Wiki/`：主题、概念、问题页与问答卡；这是默认工作层。
- `30-输出/`：文章、方案、决策记录等可复用成品。
- `90-维护报告/`：健康检查和维护日志，不作为答案的主要证据。
- `00-系统/`：规则、模板、清单和运行协议。

## 问答必经路径

```text
根目录 index.md
→ 20-Wiki/<板块>/index.md
→ 问答索引.md
→ 问题与假设/<问答卡>.md
→ 仅在核验、冲突、跨板块分析或用户要求溯源时读取 1–3 份来源笔记
```

禁止一开始全文扫描 `10-来源笔记`，禁止把低可信来源写成确定事实。Wiki 仍不足时应明确说明本地证据不足。

## 写入边界

- 原始资料只读，不移动、不改名、不修改、不删除。
- “我的理解”和“我的评价”只能由使用者填写。
- 复杂成品写入 `30-输出`；问题和风险写入 `90-维护报告`。
- 只读维护任务不得摄入或修复；只有专用增量摄入任务可在门禁通过时写入。
- 合并、改名、删除、覆盖和冲突裁决始终需要人工确认。

## 新资料与问答卡回编译

摄入 Agent 必须判断新资料对现有卡片是 `无影响/强化/补充/冲突/推翻`。只更新受影响卡片；冲突时并列证据、降低确定性、保留待核验点。只有稳定的新问题无法被现有卡承载时才新建卡片。

## 回答标准

- 先给结论，再给判断条件、例外和行动；
- 区分来源观点、可核验事实和 AI 推断；
- 时效或高风险问题必须核验当前权威来源；
- 需要回溯时只链接最相关的少量来源笔记。
